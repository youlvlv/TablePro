<Update label="June 25, 2026" description="v0.53.0">
  ### New Features

  - **Multiple tags per connection**: Assign several tags in the connection form and filter the welcome list by tag with Match Any or Match All (#744)
  - **Per-column value filter**: Click the funnel icon on a column header to pick which loaded values to show, across several columns at once. It filters loaded rows without re-querying (#1454)
  - **Elasticsearch support**: Connect to 7.x and 8.x, browse indices, run Query DSL in a console, and edit documents in the data grid. Install from Settings > Plugins (#1529)
  - **ER diagram cardinality**: Relationships show one-to-one, one-to-many, and optional variants in crow's foot notation, read from primary keys and unique indexes. Junction tables collapse into a single many-to-many link, with a toolbar toggle to expand them (#1335)
  - **Export ER diagram to SQL**: A toolbar button opens a query tab with CREATE TABLE and foreign key statements for the current schema (#1335)
  - **Oracle native network encryption**: An option in the connection form, off by default, for servers that require encrypted traffic (#1746)

  ### Improvements

  - The connection switcher and welcome list now show each connection's tags and group (#1323)
  - The ER diagram uses a more compact layout, keeps foreign-key-linked tables together, and tints each connected group with its own header color (#1755)
  - When an Oracle server drops the connection during login, the error dialog shows which handshake phase it stopped at, which helps diagnose Oracle 11g (#1746)

  ### Bug Fixes

  - Opening a new query tab now puts keyboard focus in the SQL editor instead of the sidebar filter, so you can type right away (#1765)
  - Raw filters in the data grid now work on document and key-value databases; the typed text was dropped before reaching the driver (#1529)
  - Connecting to Oracle no longer crashes on certain server values during the handshake; a bad packet now fails the connection with an error (#1746)
  - Connecting to Oracle no longer hangs when the server permits but does not require native network encryption; TablePro connects in clear text by default, like Oracle's own clients (#1746)
  - Following a foreign key into another schema now opens the correct table on SQL Server and Oracle, instead of the default schema (#1754)
  - Browsing or editing a SQL Server or Oracle table outside the default schema no longer fails with "Invalid object name" or writes to the wrong table (#1754)
</Update>

  - Container names match each database: Dataset for BigQuery, Keyspace for Cassandra and ScyllaDB (#509)

  - The inspector panel resizes freely by dragging its divider
  - Switching PostgreSQL schemas sets the search path to just the selected schema (#1662)

  - PostgreSQL and Redshift autocomplete now completes tables and columns from schemas other than the selected one (#1668)

  - Creating a table turns the Create Table tab into the new table's tab and updates the sidebar, and Cmd+S creates the table (#1664)
  - Format Query can be undone with Cmd+Z and formats only the selection when one is active (#1645, #1656)
  - Foreign key jump arrows survive sorting, filtering, and paginating, and PostgreSQL foreign keys are read from the system catalogs
  - Sorting a query result no longer overwrites the SQL editor or an open `.sql` file (#1645)
  - iCloud Sync between the iPhone and Mac apps uses the Production CloudKit environment
  - Exports no longer fail mid-table on servers with a statement time limit (#1633)

  - Large SQL scripts no longer freeze the editor; above 2 MB it suspends syntax highlighting and inline AI so typing and scrolling stay responsive (#1652)
  - Opening a query from history in the Quick Switcher loads the full query instead of a 100-character preview

  ### Security

  - Imported connections from a deep link or shared file can no longer carry a pre-connect script that runs a shell command
  - External database links ask for confirmation before connecting, and a password in the link is never saved to the Keychain
  - MCP tools enforce each connection's access level, AI policy, and token scope, and the MCP server requires a paired token by default
  - An installed plugin's code signature is re-checked right before it loads, so the binary cannot be swapped after the first check
  - MongoDB filter values in the Contains, Not Contains, Starts With, Ends With, and Regex operators can no longer inject query operators
  - On iOS, TLS certificates are validated for verify SSL modes, copied values stay on the device, and the home screen widget no longer stores database host and port on disk
